## 最简易的DEMO

Cherry Studio知识库 

- 选定文档处理 MinerU
- 选择嵌入模型和重排序模型
- 设置策略和参数，嵌入唯独、分段大小、重叠大小、匹配度阈值、检索策略、返回文档片段数目
- 添加数据源头
- 开启即可使用


## 通过DIFY等平台

Dify 就是个网页版的知识库搭建工具，点点鼠标就能用，不用写代码。想象一下你要搭个"公司文档问答机器人"，把公司的规章制度、产品手册扔进去，员工问问题就能自动回答。Dify 就是干这个的。

先装 Docker（一个打包好的软件运行环境），然后运行这几行命令：

```bash
git clone https://github.com/langgenius/dify.git
cd dify/docker
docker-compose up -d
```

等几分钟装完，浏览器打开 `http://localhost` 就能看到界面了。第一次要注册个账号，随便填。

进去之后点"创建知识库"，把你的文件拖进去就行。PDF、Word、网页链接都支持。上传完会看到几个选项：

**文档怎么切块**：AI 不能一次读完整本书，得切成小段。默认设置就行，别管它。如果文件特别长（比如几百页的法律文件），可以把分块大小调大点，比如从 500 改成 1000。

**怎么搜索**：默认是"语义搜索"，就是理解意思来搜。如果你要找特定的词（比如"第三条款"），可以开启"关键词搜索"。

配置完点"保存"，等它处理完（进度条走完），就能用了。在"应用"里创建个聊天机器人，选择刚才的知识库，就能测试问答效果。

如果回答不准确，可能是文件切得太碎了，或者搜索设置不对。Dify 有个"查询改写"功能（在高级设置里），能把口语化的问题变成标准问法，开启后准确率会提高。

数据都存在你自己电脑上，不会上传到别人服务器。




## 开源项目

RAGFlow 是国内团队做的开源知识库项目，和 Dify 类似，也有网页界面，但它的特长是"看懂"复杂文档。

比如你有扫描版的 PDF（照片拍的文件）、或者排版很乱的表格，Dify 可能识别不准。RAGFlow 专门优化了这个，能准确读取这类文件。

装 RAGFlow 也是用 Docker：

```bash
git clone https://github.com/infiniflow/ragflow.git
cd ragflow
docker-compose up -d
```

装完之后浏览器打开 `http://localhost:9380` 就能用了。

操作流程和 Dify 差不多：创建知识库 → 上传文档 → 等处理完 → 创建聊天应用。不同的是，RAGFlow 会自动识别文档里的图片、表格，把它们单独提取出来，这样问答时能更准确地引用图表内容。

RAGFlow 支持国内常用的 AI 模型（比如通义千问、智谱 GLM），如果你用的是国产大模型，RAGFlow 兼容性更好，不用折腾。

**什么时候用 RAGFlow 而不是 Dify？**

- 你的文档是扫描件或者图片
- 文档里有大量表格需要精确提取
- 你用的是国产大模型
- 需要处理多栏排版、竖排文字这类复杂格式

如果就是普通的 Word、PDF 文档，用 Dify 就够了，界面更友好。
