## 1. 常用 alias 自动生成表格

下面这张表是通过 JS 数组渲染出来的，而不是手写 HTML 表格。

export const aliases = [
  {
    name: 'gs',
    command: 'git status -sb',
    desc: '更短的 git status，日常必备',
  },
  {
    name: 'gc',
    command: 'git commit -am',
    desc: '快速提交已跟踪文件（仍建议正常写 commit）',
  },
  {
    name: 'gl',
    command: 'git --no-pager log --oneline -15',
    desc: '查看最近 15 条提交，适合 check 历史',
  },
  {
    name: 'gp',
    command: 'git push origin HEAD',
    desc: '推送当前分支到远程同名分支',
  },
]

<table>
  <thead>
    <tr>
      <th>Alias</th>
      <th>Command</th>
      <th>说明</th>
    </tr>
  </thead>
  <tbody>
    {aliases.map((a) => (
      <tr>
        <td>
          <code>{a.name}</code>
        </td>
        <td>
          <code>{a.command}</code>
        </td>
        <td>{a.desc}</td>
      </tr>
    ))}
  </tbody>
</table>

> 提示：你可以直接在自己的 shell 配置里批量粘贴这些 alias。

## 2. "一键生成" Oh My Posh 主题列表

下面这个代码块的内容，也是用 JS 拼出来的，而不是手写：

export const themes = ['paradox', 'agnoster', 'atomic', 'spaceship']

<pre>
  <code className="language-bash">
    {themes
      .map((t) => `oh-my-posh init pwsh --config ~/.poshthemes/${t}.omp.json`)
      .join('\n')}
  </code>
</pre>

你可以把这些命令复制到 PowerShell 里，快速试几个主题，然后挑一个顺眼的。

## 3. 和现有 Markdown 的关系

- 这一页是 **Client 标签**，和 `index.md` 放在同一目录 `99-manual/terminal/` 里（底层文件为 `client.mdx`）。
- `index.md` 仍然是默认标签页（Overview）；Client 只是多了一个体验向入口。
- Tab 系统通过 `tab:` frontmatter + `TabContentLayout` 自动把它们拼在一起：
  - 不需要改路由结构
  - 不会影响搜索索引（Pagefind 仍然看到完整 HTML）
  - 和纯 `.md` 完全兼容

以后如果你想在别的章节搞一个"MDX 彩蛋页"，复制这一套结构就可以了。
