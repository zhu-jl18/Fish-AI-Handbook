## 常见反模式
- 冗长却无结构；混中英；示例当事实；未声明输出格式；上下文与指令混成一坨；
- 边界未定义；无错误处理；多目标捆绑；在长对话中“继续修正”导致目标漂移。

## 好/坏对照（摘要自“好与坏”）

坏：
```text
写个Python程序
```

好：
```text
请编写 Python 函数 calculate_grade_average，输入 students_scores，
返回 {"average": number, "grade_distribution": {...}}；
处理空列表/越界分数；遵循 PEP8 与类型注解。
```

## 自检清单
- 是否单一目标且边界明确？
- 输出是否可机判（Schema/枚举/错误码）？
- 是否含 1-2 个边界样例？
- 是否声明“无答案=空结果”策略？
