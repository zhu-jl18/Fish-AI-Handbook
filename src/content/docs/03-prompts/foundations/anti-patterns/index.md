记住一句话：模型大多够聪明，提问的人不够严谨。

下面这些反模式，基本是天天能看到的垃圾 Prompt。对照你自己的聊天记录，一个个改掉。

## 1. 一句话糊弄过去

坏例：

```text
帮我写个总结。
```

改法：把任务和用途说清楚。

```text
帮我写一份给没参会同事看的会议总结，让他们知道：
1）结论是什么；
2）谁接下来要做什么。
原始记录在下面：
"""
...
"""
```

## 2. 一次干一堆事

坏例：

```text
帮我想个定价策略，顺便写一篇公告，再顺便翻成英文发推。
```

改法：拆成多步，每条消息只干一件事。要是你对人也这么说，人也会干崩。

## 3. 把所有上下文一股脑丢进来

坏例：

```text
这里是整个项目的代码和所有文档，你帮我看看为什么登录失败。
```

结果很简单：模型浪费时间在无关信息上，然后给你一堆模糊废话。

改法：只给这次任务必要的几块内容，前面标清楚是什么、时间是啥。

```text
我要排查线上登录失败。
你只需要看这些：

1）auth 服务今天 11:00–11:05 的错误日志：
"""
...
"""

2）当前线上 auth 配置：
"""
...
"""

先基于这两块给出可能原因；如果还不够，再告诉我你需要什么信息。
```

## 4. 只会说「不要」，不会说「要什么」

坏例：

```text
帮我写条朋友圈，不要太官方，不要太鸡汤。
```

这种废话，连人都执行不了，更别指望 GPT‑5.1、DeepSeek R1、Gemini 2.5 Pro、Claude 4.5 Sonnet 之类的模型猜到你脑子里在想什么。

改法：把正向要求说清楚。

```text
帮我写条朋友圈文案：
- 语气：像跟熟人说话，别装专家。
- 最长 80 字。
- 不要出现「梦想」「逆袭」「鸡汤」这些词。
```

## 5. 只说「给我 JSON」，不定义结构

坏例：

```text
用 JSON 输出学习计划。
```

然后你就会得到每次字段都不一样的垃圾结构。

改法：先写出你要的 schema，再让模型往里填。

```text
按下面这个 JSON 结构输出：

{
  "weeks": [
    {
      "week": 1,
      "theme": "本周主题",
      "tasks": ["任务 1", "任务 2", "任务 3"]
    }
  ]
}

要求：
- week 是数字，从 1 开始；
- 每周 tasks 必须有 3 条，内容要具体到「做什么」。
```

## 6. 把模型当实时搜索引擎

坏例：

```text
帮我总结某公司最新一季财报重点。
```

模型要么胡编，要么用过期数据。然后你再骂它不靠谱。

改法：自己把你关心的那几段财报贴过来，让模型做「提炼和重组」，不要让它凭空搜索。

## 7. 聊天式发牢骚，不给任何出口

坏例：

```text
我最近工作和生活都很迷茫，你说怎么办。
```

你给的是发泄，不是任务，模型当然只会回一堆鸡汤。

改法：先收敛成一个具体决策，再要「下一步动作」。

```text
我在纠结要不要在 3 个月内换工作。
下面是我的情况：
"""
...
"""

请帮我做两件事：
1）把我的主要顾虑整理成 3–5 条；
2）列出这周可以立刻做的 3 个动作（比如和谁谈、写什么清单）。
```

## 发出去前的最低限度自检

在你点发送之前，问自己五句：

1. 这条消息里是不是只有一个核心任务。
2. 完成这个任务，模型必须知道的东西，我都给了没有。
3. 我有没有说清楚想要的输出形式（段落 / 列表 / JSON / 表格）。
4. 「不要 XXX」能不能换成「要 YYY」。
5. 如果是我给人类同事下任务，我会怎么说。那就照那个写，别更烂。

做不到这些，就别怪模型蠢。
