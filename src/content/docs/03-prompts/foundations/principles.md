目标很简单：让你在用 GPT‑5.1、DeepSeek R1、Gemini 2.5 Pro、Claude 4.5 Sonnet 这些模型时，别再靠运气碰到好回答。

问题通常不在模型，在你。你提问之前几乎不想问题，只会丢一句「帮我看看」。

先讲一个最低限度的心智模型，然后给一个可以直接复用的通用骨架。

## 模型是什么，不是什么

把模型当成这样一个人：

- 脑子不差：推理、写代码、写文案都行。
- 没记性：它只看到这一次输入，你之前说过什么，它不会自动「懂你」。
- 不会读心：你不说，它就不知道你打算拿结果干嘛。

结论：如果你连任务、信息、格式和约束都说不清楚，它照样能给你一堆看起来聪明但没什么用的答案。

## 每次写 Prompt 前先想清楚的四件事

懒得背别的，就记这四个问题：

1. 这次的任务到底是什么。
   用「动词 + 对象」，比如「改写这封邮件，让它更礼貌」「解释这段代码，按模块说明」。

2. 模型完成任务必须知道什么。
   删掉和任务无关的故事。改邮件，就给原文、场景、对方是谁，够了。

3. 你要的输出长什么样。
   一段话？一封邮件？一个列表？一段 JSON？不说，就随缘。

4. 有什么约束。
   字数、语气，能不能自己脑补事实，做不到时该说什么（比如「不知道就说不知道」）。

如果你不愿意在脑子里过一遍这四条，就别抱怨输出质量不稳定。

## 一个通用骨架

下面这个骨架能覆盖大部分写作、解释、规划类任务。你可以直接保存成模板，用的时候改几行。

```md
任务：
你要完成的任务是：{一句话说明，比如「把这封邮件改写得礼貌且简洁」}。

输入：
下面是你需要用的原始内容：
"""
{粘贴原文或资料}
"""

输出格式：
请只输出以下结构，不要多输出一堆解释：

- 标题：{如果不需要就写「无」}
- 正文：{主要内容}
- 备注：{可选，没有就写「无」}

约束：
- 语气：{例如「礼貌但直接」「口语，别装专家」}
- 长度：{例如「不超过 150 字」}
- 内容要求：{例如「不要添加新的事实；信息不足就说不知道，不要瞎编」}
```

你完全可以为不同场景各复制一份：工作邮件、代码解释、学习计划，自己改几行描述就行。不要每次都从零开始生造一条烂 Prompt。

## 三个典型场景

写工作邮件：

```md
你要完成的任务：
把这封情绪化的中文邮件，改写成礼貌、专业的工作邮件。

原文：
"""
{粘贴原文}
"""

输出格式：
Subject: {英文主题}
Body:
{英文正文，合理分段}

约束：
- 语气专业，但不要跪地道歉。
- 不要承诺具体时间，只说「等我有进展会再联系」。
- 如果原文信息不足以写出合理邮件，先问我 1–2 个问题再改写。
```

解释代码：

````md
任务：
作为一名有耐心的高级工程师，帮我解释下面这段代码，假设我是刚入门。

代码：
```language
{粘贴代码}
```

输出格式：
1. 用几句话说清楚这段代码整体在干嘛。
2. 分段解释关键函数和逻辑。
3. 总结「如果我要改这个功能，最可能改哪几处」。

约束：
- 少用行话，用的话简单解释一下。
- 看到明显 bug 或坏味道就直说，不用客气。
````

做学习计划：

```md
任务：
帮我做一个 4 周的「{主题，比如 Python 入门}」学习计划。

输入：
- 目前水平：{比如「完全零基础」}
- 每周可用时间：{比如「每周 5 小时」}
- 目标：{比如「能写脚本处理 Excel」}

输出格式：
- 总体目标（1 段话）
- 第 1–4 周的计划（每周几条具体安排）
- 建议的练习题型（描述题型，而不是给网站链接）

约束：
- 每周都必须有「看什么 + 做什么」两部分。
- 禁止写「多练习」「多思考」这种废话，要具体到「做哪几类题」。
```

## 最后一句丑话

如果你提问前连「任务」「信息」「格式」「约束」这四个字都懒得想清楚，那你用再多所谓「高级 Prompt 技巧」也救不了输出质量。

先把这些基础做对，再谈花活。
