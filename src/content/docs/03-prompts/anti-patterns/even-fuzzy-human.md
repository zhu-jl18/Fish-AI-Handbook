> 结论：别指望“你懂的”“就那个”“镜像的”这类强个人化语境能被模型读懂。把“对象”“动作”“方向/范围/定位”讲清楚，哪怕多写两行，也比来回十轮强。 



## 歧义语句

TMD 我都看不懂你要说啥，你指望模型懂？新鲜出炉的真实案例：

```text
尊重原因，保持原有格式不变，用简体中文依次重写 @.claude/agents/ 和 @.claude/commands/ 下所有的 md 文件，生成新的中文 md 文件
```
What Gemini did:
```text
WritingFile: Writing to .claude\agents\codebase-analyzer.md 
```

群友们都在猜这位仁兄到底要干什么，我第一时间反应竟然和Gemini 一致，😄 哈哈。Guess What？

```text
1. cp all md files in .claude/agents/ and .claude/commands/ to {rawname}-zh.md in the same dir
2. rewrite content in all *-zh.md files to 简体中文
```

一个退化版本不需要思考的：

```text
把所有 .claude/agents 和 .claude/commands 下的 md 文件复制到同一路径下，新文件名后缀加-zh，同时将新文件内容翻译成中文，保持原有的协作格式。
```





## 症状

- 使用只有你团队/项目内才懂的简称、外号、口头禅（例如“就换成镜像的那个”）；
- 对“对象”与“动作”不做限定（哪个图标？改什么？如何改？到什么程度？）；
- 用方位/关系词但缺乏参照物（“右侧”“当前”“镜像的”没有明确指向）。

## 用只有自己懂的表达 


坏例（真实灵感案例）

用户对 Kiro（AWS AI Code Agent）说：

```text
修改 now playing 栏在播放控制按钮右侧的开关，图标改成当前镜像的
```

用户真实意图：

```text
把这个图标左右方向掉个个儿，也就是做水平镜像（镜像对称）
```

模型的误解：

```text
当前使用的是 FiLayout 图标，我理解你是想使用“当前正在播放的歌曲的镜像专辑封面”作为这个开关的图标
```

背景：

- 用户想用 Kiro 做一个类似 Spotify 的应用，运行在自建私有服务器上；
- 语句中“当前”“镜像的”在项目上下文里没有被界定到“水平翻转图标”，导致模型把“镜像”联想到“专辑封面镜像”。

## 结果

- 实现方向跑偏，改到完全不是你要的点；
- 辅助 agent 进一步“合理化”错误假设，越走越远；
- 你以为模型“读不懂话”，其实是你没给足信息。

## 改法（把对象/动作/方向讲清楚）

最小可行做法：写清楚 4 件事。

1) 对象：具体到组件/变量/文件/选择器；
2) 动作：替换/重命名/水平镜像/垂直镜像/旋转多少度；
3) 方向/范围：左右/上下/针对单个图标还是整组；
4) 约束：不改尺寸/不改颜色/兼容深色主题等。

### 好例（同一需求的清晰表述）

```text
任务：对 now-playing 栏中，播放控制按钮右侧的“开关图标”进行水平镜像（左右翻转），其他属性保持不变。

输入：
- 目标组件：NowPlayingControls/ToggleIcon（文件：components/now-playing/ToggleIcon.tsx）
- 当前图标：FiLayout（size=18, color=inherit）

输出格式：
1. 变更说明（修改了哪些文件/属性）
2. 代码差异（diff）
3. 快速回退指引

约束：
- 仅做水平镜像；不改变图标大小、颜色和位置；
- 深浅色主题下外观保持一致；
- 如果库不支持镜像，请给出 2 种可选实现：CSS transform 与 SVG 路径替换。
```

### 扩展示例（给 agent 更强的可执行性）

```text
如果直接水平镜像 FiLayout 不方便，可选方案：
A) 在 ToggleIcon 外层包一层样式：transform: scaleX(-1);
B) 用等价的左右相反图标（请列出 2 个备选）并保持 size/color；
完成后请附上对比截图与回归清单（点击态、hover、深色主题）。
```

### 模板（可直接复用）

```md
任务：
请对 {对象/组件/文件} 执行 {动作}，方向/范围为 {左右/上下/角度/单个/整组}。

输入：
- 位置/文件：{路径}
- 当前状态：{图标名/样式/数据}

输出格式：
1) 变更说明
2) 代码 diff
3) 回退步骤

约束：
- {不改尺寸/颜色/布局}
- {主题兼容/无可访问性回退则提示}
- {如库不支持，给 2 种替代实现}
```

> 原则：把“你脑子里的参照物”落在文字里。对象、动作、方向、范围、约束，填全就不歧义。

## 语义模糊的三大来源（合并：个性化表述 + 口头禅 + 不明确指代）

1) 个性化表述/团队黑话：
- 现象：使用只有本团队懂的简称、外号（“就用镜像那个”）。
- 修法：建立 Glossary，给唯一名称/路径/ID；禁止昵称。

2) 口头禅/形容词：
- 现象：用“顺手点、正式点、轻一点、适当优化”等不可量化词。
- 修法：将形容词转为可验证约束（字数/层级/禁用词/样式开关）。

3) 不明确的指代与多义词：
- 现象：它/那个/当前、镜像/版本/封面等多义词无参照物。
- 修法：给参照（截图/坐标/DOM 路径/组件路径），指定方向/范围（左右/上下/单个/整组）。

整合模板（增强版）
```md
Glossary
- {对象名}: {路径/ID}

任务：对 {对象名} 执行 {动作}，方向/范围 {左右/上下/角度/单个/整组}。
输出结构：{…}
约束：{长度/禁用词/主题/尺寸/颜色…}
不足时：先问 1–2 个澄清问题再执行。
```
